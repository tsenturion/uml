
# Экзаменационное задание

## Тема

Моделирование UML-диаграмм для информационной системы с использованием **Mermaid** и **PlantUML**

## Предметная область

**Информационная система онлайн-кинотеатра**

Система предназначена для:

* регистрации и аутентификации пользователей;
* просмотра каталога фильмов;
* оформления подписки;
* просмотра фильмов;
* администрирования контента.

## Задание

### Общие требования

* Все диаграммы **создаются в виде кода**
* Диаграммы должны соответствовать UML-семантике
* Диаграммы должны описывать **одну и ту же предметную область**
* Использовать **Mermaid только там, где он подходит**, остальные диаграммы — **только через PlantUML**

## Часть 1. Диаграммы, реализуемые с помощью **Mermaid**

Создать следующие UML-диаграммы **в формате Mermaid**:

### 1. Диаграмма последовательности

**Сценарий:**
Пользователь выбирает фильм → система проверяет подписку → начинается воспроизведение.

Должны быть:

* пользователь
* клиентское приложение
* сервер
* сервис подписок
* условие (есть подписка / нет подписки)


### 2. Диаграмма классов

Описать основные сущности системы:

Минимальный набор классов:

* User
* Movie
* Subscription
* Payment
* Admin

Отразить:

* атрибуты
* методы
* связи (ассоциации, наследование)


### 3. Диаграмма состояний

**Объект:** Подписка пользователя

Состояния, например:

* Неактивна
* Активна
* Приостановлена
* Истекла

Показать переходы между состояниями и события, которые их вызывают.


### 4. Диаграмма деятельности (возможна упрощённая реализация)

**Процесс:** Оформление подписки

Должны быть:

* последовательность действий
* ветвление (успешная / неуспешная оплата)
* конечное состояние

⚠ Допускается использование `flowchart` как аналога activity diagram.


## Часть 2. Диаграммы, реализуемые с помощью **PlantUML**

Создать следующие UML-диаграммы **в формате PlantUML**:


### 5. Диаграмма вариантов использования (Use Case Diagram)

Акторы:

* Гость
* Пользователь
* Администратор

Варианты использования:

* Регистрация
* Вход в систему
* Просмотр каталога
* Просмотр фильма
* Управление фильмами

Использовать:

* include
* extend
* границу системы


### 6. Диаграмма кооперации (Communication Diagram)

**Сценарий:**
Процесс воспроизведения фильма.

Отразить:

* объекты
* связи между ними
* нумерацию сообщений


### 7. Диаграмма компонентов (Component Diagram)

Компоненты, например:

* Web Client
* Backend API
* Auth Service
* Payment Service
* Media Storage

Показать:

* зависимости
* взаимодействие компонентов


### 8. Диаграмма развертывания (Deployment Diagram)

Отразить:

* клиентское устройство
* сервер приложений
* сервер базы данных
* облачное хранилище видео

Показать размещение компонентов на узлах.


## Подсказки по ключевым частям

* Для Mermaid:

  * строго соблюдать синтаксис (`sequenceDiagram`, `classDiagram`, `stateDiagram`)
  * не пытаться моделировать компоненты и deployment
* Для PlantUML:

  * использовать специализированные ключевые слова (`actor`, `usecase`, `component`, `node`)
  * не смешивать разные типы диаграмм в одном файле
* Все диаграммы должны логически **соответствовать одной системе**


## Что проверить перед отправкой (чек-лист)

* [ ] Все диаграммы созданы кодом, а не рисунками
* [ ] Mermaid используется **только** для поддерживаемых диаграмм
* [ ] PlantUML используется для остальных UML-диаграмм
* [ ] Нет синтаксических ошибок
* [ ] Названия элементов осмысленные и связаны с предметной областью
* [ ] Диаграммы не противоречат друг другу


## Советы по улучшению работы

* Используйте единый стиль именования (CamelCase или snake_case)
* Не перегружайте диаграммы — лучше несколько простых, чем одна перегруженная
* Добавляйте комментарии в код диаграмм
* Сначала продумайте логику системы, **потом пишите код**
* Проверяйте рендеринг диаграмм до сдачи
